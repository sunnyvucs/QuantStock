📄 PROJECT DOCUMENT

🧠 AI Stock Insights Platform (India-Focused)

⸻

1. 📌 PROJECT OVERVIEW

1.1 Objective

Build a scalable web platform that:
	•	Provides AI-based stock trend insights
	•	Handles high traffic efficiently
	•	Generates revenue via ads + premium features
	•	Uses precomputed predictions instead of real-time heavy computation

⸻

1.2 Problem Statement

Current system:
	•	Runs predictions per request (session-based)
	•	No data persistence
	•	Not scalable
	•	No SEO or monetization capability

Target system:
	•	Precomputed predictions
	•	Fast API responses
	•	Search engine optimized pages
	•	Monetizable platform

⸻

1.3 Success Criteria

Metric	Target
API Response Time	< 200 ms
Monthly Users	50K+
Ad Revenue	₹20K+/month
System Uptime	99%


⸻

2. 🏗️ SYSTEM ARCHITECTURE

2.1 High-Level Architecture

Frontend (React)
        ↓
Backend API (FastAPI)
        ↓
Redis Cache
        ↓
PostgreSQL Database
        ↓
ML Model Service
        ↓
Data Pipeline (CRON Jobs)


⸻

2.2 Architecture Principles
	•	No heavy computation on user request
	•	Precompute + cache everything
	•	Modular services
	•	Scalable and cloud-ready

⸻

3. 🧩 SYSTEM COMPONENTS

⸻

3.1 Frontend

Tech Stack:
	•	React (or Next.js for SEO)

Features:
	•	Stock search
	•	Prediction display
	•	Chart visualization
	•	SEO pages

⸻

3.2 Backend API

Tech Stack:
	•	FastAPI (Python)

Responsibilities:
	•	Serve prediction data
	•	Validate stock inputs
	•	Handle caching
	•	Provide REST APIs

⸻

3.3 Database

Tech:
	•	PostgreSQL

Tables:

predictions

id SERIAL PRIMARY KEY
stock VARCHAR(10)
timeframe VARCHAR(10)
trend VARCHAR(10)
probability FLOAT
confidence VARCHAR(10)
created_at TIMESTAMP


⸻

3.4 Cache Layer

Tech:
	•	Redis

Purpose:
	•	Reduce DB calls
	•	Serve fast responses

⸻

3.5 ML Model Service

Responsibilities:
	•	Load trained models
	•	Generate predictions
	•	Return structured output

⸻

3.6 Data Pipeline

Trigger:
	•	CRON Jobs (every 1 hour)

Responsibilities:
	•	Fetch stock data
	•	Run predictions
	•	Store results in DB
	•	Update cache

5. 🤖 MACHINE LEARNING DESIGN

⸻

5.1 Input Features
	•	Price (OHLC)
	•	Volume
	•	Technical indicators (RSI, MACD, Moving Averages)
	•	Sentiment score (optional)

⸻

5.2 Models Used
	•	Gradient Boosting (XGBoost)
	•	LSTM (for time series)

⸻

5.3 Output Format

{
  "trend": "UP/DOWN",
  "probability": 0.65,
  "confidence": "MEDIUM"
}


⸻

5.4 Model Strategy
	•	Do NOT predict exact price
	•	Predict probability of movement
	•	Use ensemble approach
